MariaDB安装和部署

2019-02-28
  • MySQL

“””
MariaDB是MySQL的分支最大的特点就是采用XtraDB存储引擎，并和原生版本兼容，
保留了Mysql的自由开放，XtraDB是Percona开发的高性能存储引擎，主要是为了替代InnoDB.
除了XtraDB引擎，后起之秀TokuDB支持高性能、事务处理的存储引擎，性能比InnoDB高出很多，
超高的insert性能、高压缩比、支持在线修改索引、添加字段。安装时Mysql、MariaDB、Percona Server 不能共存，
要部署MariaDB就必须卸载另外两个。否则出现“mysql service masked”错误。
“””

安装MariaDB数据库(Debian系列系统适用，包括Ubuntu和Deepin)
$ sudo aptitude install -y mariadb-server mariadb-client

安装数据库实例
$ sudo mysql_secure_installation

终端提示和输入：

设置root密码

Enter current password for root (enter for none): <–初次运行直接回车

设置密码

Set root password? [Y/n] <– 是否设置root用户密码，输入Y并回车或直接回车
New password: <– 设置root用户的密码
Re-enter new password: <– 再输入一次你设置的密码

其他配置

Remove anonymous users? [Y/n] <– 是否删除匿名用户，Y
Disallow root login remotely? [Y/n] <–是否禁止root远程登录,n
Remove test database and access to it? [Y/n] <– 是否删除test数据库，Y
Reload privilege tables now? [Y/n] <– 是否重新加载权限表，Y

数据库设置相关操作

启动数据库

$ sudo systemctl start mysql

查看数据库服务状态（MariaDB与其他Mysql数据库不能共存在同一操作系统）

$ sudo systemctl status mysql

设置mysql随系统服务启动

$ sudo update-rc.d mysql defaults

撤销随系统服务启动

$ sudo update-rc.d -f mysql remove

重启数据库服务，使配置生效

$ sudo systemctl restart mysql

初始化完成MariaDB后,就可以测试登录了
”“” 之前版本mysql不同，需要获得操作系统管理员权限，才能登录MariaDB的root用户
普通操作系统用户不能登录MariaDB数据库root用户 “”“
$ sudo mysql -u root -p

创建普通数据库用户 （登录普通数据库用户则不需要获得操作系统管理员权限）
MariaDB [(none)]> create user ‘user_name‘@’host_name’ identified by ‘password’;

“””CREATE USER 等账户管理语句中的账户由一个用户名和一个主机名组成，其格式为‘user_name’@’host_name’。
主机部分是指客户从哪些主机去连接服务器。如若限制某个用户从指定的主机连接 MariaDB 服务:

MariaDB [(none)]> CREATE USER 'yangfan'@'localhost' IDENTIFIED BY 'password';
MariaDB [(none)]> CREATE USER 'yangfan'@'169.254.195.91' IDENTIFIED BY 'password';
第一条语句，创建一个名为 yangfan 的用户，并限制其只可从本机进行连接操作；
第二条语句，创建一个名为 yangfan 的用户，并限制其只可从 ip 169.254.195.91 的客户机上连接服务。
虽然用户名相同，实际上，他们并不是同一个账户，其对应的权限也可能不尽相同。

当然很多情况下并不需要严格要求用户只能从一台主机连接服务器。这个时候就可以用上通配符：

MariaDB [(none)]> CREATE USER 'yangfan'@'169.254.195.%' IDENTIFIED BY 'yangfan';
MariaDB [(none)]> CREATE USER 'yangfan'@'%.yangfanweb.cn' IDENTIFIED BY 'yangfan';
MariaDB [(none)]> CREATE USER 'yangfan'@'%' IDENTIFIED BY 'yangfan';
SQL 中 LIKE 的通配符 % 及 -，在此都可使用。如若使用 “%” 或 “-” 本身，则需使用 “\” 对其转义。

上面第一条语句创建一个名为 yangfan 的用户，并允许其在 169.254.195.* ip 段下的所有主机连接服务器；
第二条语句创建一个名为 yangfan 的用户，并允许其能够从 yangfanweb.cn 域中的任意一台主机连接服务器；
第三条语句创建了一个名为 yangfan 的用户，并允许其可从任意一台主机来连接服务器。（第三种方式最为方便，但对系统最不安全。）

“””

删除数据库用户
MariaDB [(none)]> drop user 用户名@’%’;

修改MariaDB配置文件，监听外网访问(需要重启服务才可以生效)
$ sudo vim /etc/mysql/mariadb.conf.d/50-server.cnf

Instead of skip-networking the default is now to listen only on

localhost which is more compatible and is not less secure.

bind-address = 0.0.0.0

或者注释掉这一行

#bind-address = 127.0.0.1

登录远程数据库(需要username@’%’)
$ mysql -h host_name -u user_name -p password

数据库用户权限操作

授予用户权限

MariaDB [(none)]> GRANT ALL ON test.* TO ‘yangfan‘@’localhost’ WITH GRANT OPTION;
“”” 上面语句将数据库test及其里面所有对象的所有权限授权于账户 ‘yangfan’@’localhost’
要想把权限授予其他用户，首先发出授权的用户必须具备该权限，而且必须具备 GRANT OPTION 权限。
MariaDB 允许在数据库系统全局、数据库、数据表、数据列等多种级别上进行授权。权限级别由 ON 子句控制。
“””

在使用Grant命令授权用户后，应该要使用flush privileges命令刷新权限列表

查看用户权限

MariaDB [(none)]> SHOW GRANTS FOR ‘yangfan‘@’localhost’;

更改用户权限

MariaDB [(none)]> GRANT SELECT,INSERT,UPDATE ON test.* TO ‘yangfan‘@’127.0.0.1’;

上面的语句将账户 ‘yangfan’@’127.0.0.1’ 的权限更改为，只有对数据库 test 及其里面所有对象进行 SELECT、INSERT 和 UPDATE 操作的权限。

查看结果

MariaDB [(none)]> SHOW GRANTS FOR ‘yangfan‘@’127.0.0.1’;

撤销用户权限

撤销用户授权用 REVOKE 语句，下面是 REVOKE 语句的语法：

REVOKE preivileges [ columns ] ON what FROM account;

之前我们将全部权限授予账户 ‘yangfan’@’localhost’， 现在我们对该账户进行权限的删除：

MariaDB [(none)]> REVOKE ALL ON test.* FROM ‘yangfan‘@’localhost’;

上面语句可以把账户 ‘yangfan’@’localhost’ 对数据库 test 的权限全部删除。

查看结果：

MariaDB [(none)]> SHOW GRANTS FOR ‘yangfan‘@’localhost’;

注意：USAGE 便是下文提到的“无权限”的权限。

我们再删除账户 ‘yangfan’@’127.0.0.1’ 的 INSERT 和 UPDATE 的权限：

MariaDB [(none)]> REVOKE INSERT, UPDATE ON test.* FROM ‘yangfan‘@’127.0.0.1’;

查看结果：

MariaDB [(none)]> SHOW GRANTS FOR ‘yangfan‘@’127.0.0.1’;

可以看到现在‘yangfan‘@’127.0.0.1’这个用户只有SELECT权限

可以授予用户的权限由很多种。下面3张表即为权限的汇总。

配置MariaDB的字符集：
“”” MySQL在5.5.3版本之后增加了utf8mb4，向下兼容utf8。MariaDB的10.0版本之后
默认使用utf8mb4作为Debian系列系统的默认字符集编码。utf8mb4的优点：支持最大字符长度为4字节。
缺点：存储对多消耗一些空间，对于 CHAR 类型数据，utf8mb4 会多消耗一些空间，
根据 Mysql 官方建议，使用 VARCHAR 替代 CHAR。
“””

修改字符集配置

设置客户端：

$ sudo vim /etc/mysql/mariadb.conf.d/50-mysql-clients.cnf

[mysql]
default-character-set = utf8
“”” 这个命令会讲下列设置修改为utf编码：
character_set_client
character_set_connection
character_set_results
collation_connection
相当于进入到mysql终端进行输入：
MariaDB [(none)]> SET NAMES ‘utf8’;
“””

进入数据库查看修改结果

设置服务端

$ sudo vim /etc/mysql/mariadb.conf.d/50-server.cnf

[mysqld]
character_set_server = utf8
collation-server = utf8_general_ci

创建数据库的时候指定编码格式

MariaDB [(none)]> create database db_name character set utf8;

创建表的时候指定编码格式

MariaDB [(none)]> create table db_name(
-> id int(11),
-> name varchar(50) character set utf8,
-> class_time int(11),
-> PRIMARY KEY(id)
-> )DEFAULT CHARSET=utf8;

对已经创建完成的表或者数据库通过ALTER进行修改

修改某一个数据库

MariaDB [(none)]> ALTER database db_name character set utf8;

修改某一个表

MariaDB [(none)]> ALTER table table_name character set utf8;

对单个编码设置进行设置，如：character_set_database

MariaDB [(none)]> set character_set_database=utf8;
Query OK, 0 rows affected (0.00 sec)

查看数据库版本
$ mysql -V
